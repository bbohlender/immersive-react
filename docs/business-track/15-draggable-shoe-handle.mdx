To make the shoe component draggable we create a `Draggabe` component, which we can wrap around the shoe to make it draggable.
First, we define `positionOffset` as the offset between the draggable handle to the wrapped object.

```ts
const positionOffset = new Vector3(1.4, 0, 0);
```

Next, we create the `matrixHelper`, which we for storing a matrix calculation later on

```ts
const matrixHelper = new Matrix4();
```

Next, we define the Draggable component with a group wrapping everything, a ref to modify the group's position, and a ref to store the `draggingPointerId`, which is the id of the pointer that is currently dragging the Draggable component.

```tsx
function Draggable({ children }: { children?: ReactNode }) {
  const ref = useRef<Group>(null);
  const draggingPointerId = useRef<number | undefined>(undefined);
  return (
    <group ref={ref}>
      {children}
      <mesh position={positionOffset}>
        <sphereGeometry />
        <meshPhongMaterial color="grey" />
      </mesh>
    </group>
  );
}
```

Next, we implement the pointer event handlers on the `mesh` in the `Draggable` component.
First, the `onPointerDown` event handler checks if a dragging is already in progress by checking `draggingPointerId.current`. It then stores the pointer id of the pointer that started the interaction and sets the pointer capture to ensure we keep receiving events even if the pointer moves.

```tsx
onPointerDown={(e) => {
  if (draggingPointerId.current != null) {
    return;
  }
  draggingPointerId.current = e.pointerId;
  (e.target as HTMLElement).setPointerCapture(e.pointerId);
}}
```

Next, we implement the `onPointerMove` handler that does the actual dragging calculation. For this calculation we want to use the point of the interaction (`e.point`) and subtracting the offset to the draggable object. The problem is that `e.point` is the position relative to the global coordinate system while the position and `ref.current.position` is relative to the coordinate system of the draggable's parent. Therefore we calculate the `matrixHelper` as the inverse of the parent's world matrix allowing us to transform the interaction point (`e.point`) into the local space of the draggable's parent.

```tsx
onPointerMove={(e) => {
  if (draggingPointerId.current == null || ref.current == null) {
    return;
  }
  matrixHelper.copy(ref.current.parent!.matrixWorld).invert();
  ref.current.position
    .copy(e.point)
    .applyMatrix4(matrixHelper)
    .sub(positionOffset);
}}
```

Next, we implement the `onPointerUp` handler to clean up the `draggingPointerId` when dragging ends:

```tsx
onPointerUp={() => {
  if (draggingPointerId.current == null) {
    return;
  }
  draggingPointerId.current = undefined;
}}
```

Finally, we use the `Draggable` component by wrapping the `Model` and `ColorPicker` components in `App.tsx` with it.

```tsx
<group position={[0, 1.1, -0.3]} scale={0.2}>
  <Draggable>
    <Model />
    <ColorPicker />
  </Draggable>
</group>
```
